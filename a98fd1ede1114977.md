# CSUTKG Resource Index

CSUTKG Resource Index 是一个面向技术研究、学术文献整理与知识图谱构建的轻量级资源外链汇总工具。该项目定位为半自动化的 Markdown 资源索引系统，主要服务于需要批量管理、分类归档和快速检索大量外部技术链接的研究人员、开发者以及文档维护者。

与通用的书签管理工具或笔记软件不同，CSUTKG Resource Index 专注于将无结构的 URL 列表转化为具备层级关系、可版本化、可协作维护的 Markdown 文档集合。项目本身不存储具体内容，而是通过规范的文档结构和元数据注释，帮助团队或个人建立清晰的外部资源引用体系，解决链接散落、上下文丢失、重复整理等常见问题。当前版本为第 14 批资源整合批次，累计收录 180 个外部资源链接。

## 功能概览

**批量 URL 导入与解析** 支持从文本文件或标准输入中批量读取原始 URL 列表，自动识别并过滤无效格式，生成初始化的 Markdown 条目模板。

**分层目录树生成** 依据资源类型、所属领域或来源仓库，自动构建多级分类目录，并为每个条目生成带有时间戳和批次编号的唯一锚点标识。

**元数据标注与维护** 允许为每个外链添加标签、简要说明、归属类别以及关联批次号，所有元数据以标准 Markdown 注释或列表缩进形式存储，保持文档纯净可读。

**状态标记与进度跟踪** 支持对每个链接标记待处理、已查阅、需更新或已失效等状态，并生成汇总统计报表，便于团队协作时同步进度。

**变更历史记录** 每次新增或修改资源时，自动在文档尾部追加操作日志，记录操作人、操作时间和具体变更内容，满足审计追溯需求。

**纯 Markdown 与 Git 原生集成** 所有索引数据以纯 Markdown 文件存储，天然支持 Git 版本控制，可利用分支、差异对比、合并请求等标准工作流进行协同维护。

## 应用场景

技术团队内部知识库构建：开发团队可将日常调研中遇到的第三方库文档、技术博客、API 参考、规范标准等散落链接统一收录到 CSUTKG Resource Index 中，按项目或技术栈分类，新成员入职时可快速获取体系化的学习路径参考。

学术文献与数据集整理：研究人员在撰写综述或进行实验对比时，需要收集大量论文主页、代码仓库、数据集下载地址和工具官网。使用本项目可建立结构化的文献索引库，并为每条记录添加领域标签和阅读状态，大幅提升文献调研效率。

开源项目外部依赖链管理：开源软件维护者常常需要在 README 或文档中引用上游依赖、相关工具或社区扩展列表。CSUTKG Resource Index 可作为外部链接的中转整理站，先在本项目中完成链接去重、分类和验证，再将生成的 Markdown 片段嵌入到最终文档中。

多批次资源归档与发布：对于需要周期性发布资源汇总清单的场景（如月刊、周报、技术雷达），运维人员可利用本项目的批次管理功能，将不同批次的链接按时间维度归档，并通过自动生成的索引表格快速产出可发布的文档版本。

## 快速开始

以下步骤帮助您在本地环境快速部署并运行 CSUTKG Resource Index 的基础功能。

```bash
# 克隆项目仓库
git clone https://github.com/warriorjacketenact/csutkg.git

# 进入项目目录
cd csutkg

# 安装依赖（项目基于 Python 3 开发，依赖标准库及 PyYAML 用于配置解析）
pip install pyyaml

# 运行索引构建脚本，处理当前批次资源
python scripts/build_index.py --batch 14 --input resources/batch_14.txt --output docs/index_14.md
```

执行上述命令后，脚本将读取 `resources/batch_14.txt` 中存放的原始 URL 列表，按照 `config/schema.yaml` 中定义的分类规则生成一份结构化的 Markdown 索引文档，并输出至 `docs/index_14.md`。用户可直接查看该文件，也可将其作为子文档嵌入到更大的文档集合中。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.8 及以上版本 | 是 | 核心脚本运行环境，用于解析 URL、生成目录树和统计信息 |
| PyYAML 5.4 及以上版本 | 是 | 用于读取项目配置文件，包括分类映射、元数据模板和输出格式设置 |
| Git 2.20 及以上版本 | 是 | 用于版本控制及自动提交记录，非运行时强制依赖但强烈建议安装 |
| GNU Make 3.81 及以上版本 | 否 | 提供便捷命令封装，如 make init、make build、make clean 等简化操作 |
| pandoc 2.9 及以上版本 | 否 | 可选依赖，用于将生成的 Markdown 索引转换为 HTML、PDF 或 Docx 格式用于发布 |
| shellcheck 0.7 及以上版本 | 否 | 仅当开发者需要修改辅助 shell 脚本时用于静态检查 |

## 项目结构

项目目录采用模块化分层设计，核心脚本与资源配置分离，便于独立维护和扩展。

```
csutkg/
├── README.md                         # 项目总览文档（当前文件）
├── LICENSE                           # MIT 许可证文本
├── Makefile                          # 提供 make init/build/clean 等快捷命令
├── config/                           # 配置目录
│   ├── schema.yaml                   # 定义资源分类规则、标签白名单及输出模板
│   ├── categories.d/                 # 分类子配置片段，按领域拆分便于协作
│   │   ├── algorithms.yaml           # 算法相关关键词及正则匹配规则
│   │   ├── infrastructure.yaml       # 基础设施与运维关键词规则
│   │   └── references.yaml           # 通用技术参考分类规则
│   └── templates/                    # Markdown 输出模板
│       ├── index_template.md         # 索引文档主体框架
│       └── entry_template.md         # 单条资源条目标记格式
├── resources/                        # 原始资源数据目录
│   ├── batch_14.txt                  # 当前批次原始 URL 列表（纯文本，每行一个链接）
│   └── batch_history/                # 历史批次归档
│       ├── batch_01.txt              # 第一批次原始数据
│       └── ...                       # 其他批次文件
├── scripts/                          # 可执行脚本目录
│   ├── build_index.py                # 主构建脚本，读取配置与原始数据生成索引文档
│   ├── validate_urls.py              # URL 格式校验工具，检查重复项与协议合规性
│   ├── generate_stats.py             # 统计生成器，输出分类计数与状态分布
│   └── utils/                        # 公共工具函数模块
│       ├── markdown_helpers.py       # Markdown 表格、列表生成辅助函数
│       └── yaml_loader.py            # 带缓存机制的 YAML 配置加载器
├── docs/                             # 生成的索引文档及用户手册
│   ├── index_14.md                   # 第 14 批次最终生成的索引文档
│   ├── getting_started.md            # 入门指南
│   ├── categorization.md             # 分类规则说明
│   └── deployment_examples.md        # 自动化部署示例
├── tests/                            # 单元测试与集成测试
│   ├── test_build.py                 # 构建脚本的单元测试
│   ├── test_validate.py              # URL 校验器测试
│   └── fixtures/                     # 测试用固定样本数据
│       └── sample_urls.txt           # 用于测试的小规模 URL 样本
└── .github/                          # GitHub 社区文件
    └── workflows/
        └── build_docs.yml            # GitHub Actions 自动构建工作流，每次 push 触发索引更新
```

## 贡献指南

我们欢迎并鼓励开发者以多种方式参与 CSUTKG Resource Index 项目的改进。请遵循以下步骤进行贡献。

第一步，Fork 本仓库并在本地克隆您的 Fork 副本，同时添加上游远程仓库以便同步主分支的更新。

第二步，在 resources/ 目录下新增或修改批次文件，确保每行一个 URL，并移除重复项。若需调整分类规则，请修改 config/schema.yaml 或对应的 category.d/ 子文件，并在修改说明中注明变更原因。

第三步，运行本地构建脚本验证您的更改是否产生预期输出。执行 make test 可运行全部单元测试套件，执行 make build 可基于当前配置重新生成所有索引文档。若测试不通过，请根据错误日志调整修改内容。

第四步，提交变更并推送至您的 Fork 仓库。提交信息请采用约定式提交格式，例如 feat: add new batch 15 links 或 fix: update regex for infrastructure category。随后在 GitHub 上向本仓库主分支发起 Pull Request。

第五步，等待项目维护者审阅。审阅过程中可能会要求您补充说明或进行小幅调整。合并后您的变更将自动触发 GitHub Actions 工作流，重新构建并部署最新的索引文档。

## 常见问题

问：如何处理 URL 列表中的重复链接？

答：构建脚本默认启用去重逻辑。当检测到重复 URL 时，脚本会在控制台输出警告信息，并只保留首次出现的条目。如需保留所有出现位置，可在 config/schema.yaml 中将 deduplicate 字段设为 false。但建议保持默认开启，以维持索引的整洁性。

问：能否将生成的索引文档自动发布为网页或 PDF？

答：可以。项目提供了可选的 pandoc 集成路径。您可以在安装 pandoc 后，执行 make publish-html 或 make publish-pdf 命令，构建脚本将调用 pandoc 将 Markdown 索引转换为 HTML 或 PDF 格式。输出文件默认存放于 docs/publish/ 目录下。若需自定义样式，可修改 config/templates/style.css 或相应的 pandoc 模板文件。

问：如何为已有批次追加新的链接，而不重建整个索引？

答：您可以在 resources/batch_14.txt 文件末尾追加新链接，然后运行 scripts/build_index.py --batch 14 --append 命令。该模式会读取现有索引文档，仅将新增的链接插入到对应的分类章节中，并更新统计信息，而不会影响已有条目的元数据。该操作同样会记录变更历史。
