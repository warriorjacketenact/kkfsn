# CSU TKG External Reference Index

CSU TKG External Reference Index 是一个面向技术文档维护者、开源项目贡献者以及知识库管理员的纯静态外链汇总与结构化导航工具。该项目不存储任何实际内容，仅对分散在多个仓库、文档站点以及技术社区中的外部资源进行哈希索引与分类整理，帮助团队在大规模文档集中快速定位特定主题的参考链接。

该项目定位为技术资源的外链索引层，适用于需要频繁查阅外部规范、提案、教程或代码示例的场景。通过统一的条目标识与分类标签，用户可以在不改变原有文档结构的前提下，建立跨仓库、跨站点的引用网络，降低信息查找的边际成本。

## 功能概览

**按主题分类的外链索引** 项目将收录的所有外部资源按照技术领域、文档类型或使用场景进行逻辑分组，每个分组下包含若干条目，每个条目均以固定长度的哈希标识符作为唯一键值，便于跨文档引用与版本追踪。

**纯静态的 Markdown 文档结构** 所有索引条目均以 Markdown 格式存储于仓库的 main 分支，无需数据库或后端服务即可直接浏览，兼容 GitHub、GitLab 等主流代码托管平台的在线渲染。

**基于文件名的条目定位机制** 每个资源条目对应一个独立的 .md 文件，文件名即为该条目的哈希标识符，用户可以通过标识符快速在仓库目录中定位到对应的外链记录。

**可扩展的条目元数据字段** 每个索引条目文件包含原始 URL、收录时间、所属分类、简要描述以及关联标签等元数据，方便后续进行自动化筛选或导出。

**与 CI/CD 流程集成的校验能力** 项目提供脚本用于检测失效链接或 URL 格式异常，可在持续集成流程中自动运行，确保索引库的长期可用性。

**支持批量导入与去重合并** 提供工具函数，支持从 CSV 或 JSON 格式的数据源批量导入外部链接，并基于 URL 哈希自动去重，避免重复收录。

**轻量级本地预览服务** 内置基于 Python 或 Node.js 的简易 HTTP 服务器，用户可在本地启动预览环境，无需配置复杂的前端构建工具即可查看索引全貌。

## 应用场景

**技术文档团队的参考链接管理** 当文档团队需要统一管理分布在数十篇技术文章中的外部引用时，可使用该项目建立独立的外链索引库。每篇文章只需引用索引条目的标识符，后续如需更新外部 URL，仅需修改索引条目中的原始链接，无需逐一修改所有文档。

**开源项目 README 与 Wiki 的外链聚合** 开源项目维护者可将项目文档中频繁引用的规范、RFC、教程视频或社区讨论帖统一收录至该索引，在主 README 中仅保留索引仓库的链接，从而减少主文档的冗长外链列表，提升可读性。

**技术培训课程的材料整理** 培训讲师可将课程中涉及的课外阅读资料、实验环境入口、参考代码仓库等外部链接按课时或主题整理为多个索引条目，学员可通过标识符快速获取对应的学习资源，无需手动输入长 URL。

**知识库的跨仓库引用中介** 对于拥有多个代码仓库或文档站点的组织，该索引可作为跨仓库引用的中介层。不同仓库的文档均通过统一的标识符引用外部资源，避免因仓库迁移或站点改版导致的大量死链问题。

## 快速开始

以下命令演示如何将本索引仓库克隆至本地，安装基础依赖，并启动本地预览服务。

```bash
git clone https://github.com/warriorjacketenact/csutkg.git
cd csutkg
pip install -r requirements.txt
python -m http.server 8000 --directory .
```

执行上述命令后，在浏览器中访问 http://localhost:8000 即可查看索引页面的本地渲染效果。若需要执行链接校验，可运行以下命令：

```bash
python scripts/check_links.py --source ./main
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.8 或更高版本 | 是 | 用于运行链接校验脚本和本地预览服务 |
| Git 2.25 或更高版本 | 是 | 用于克隆仓库和管理版本历史 |
| pip 21.0 或更高版本 | 是 | 用于安装 Python 依赖包 |
| requests 库 2.28 或更高版本 | 否 | 仅在执行链接校验时需要，用于发送 HTTP 请求 |
| markdown 库 3.4 或更高版本 | 否 | 仅在生成索引统计报告时需要，用于解析 Markdown 元数据 |
| Node.js 18 或更高版本 | 否 | 若使用基于 Node.js 的预览服务器替代 Python 版本时可选 |

## 文档导航

| 层面 | 目录 | 回答的问题 |
|------|------|------------|
| 索引概览 | /main/README.md | 项目整体介绍、收录范围说明、分类体系定义 |
| 条目存储 | /main/{hash}.md | 每个条目的具体外部链接、收录日期、分类标签与简短摘要 |
| 校验工具 | /scripts/check_links.py | 如何检测失效链接、如何生成链接状态报告 |
| 导入导出 | /scripts/import_from_csv.py | 如何批量添加新条目、如何将索引导出为其他格式 |

## 贡献指南

提交新索引条目时，请先在 main 目录下查找是否已存在相同目标 URL 的条目，避免重复收录。若确认该链接未被收录，请按照既有条目的元数据格式创建一个新的 .md 文件，文件名使用 16 位十六进制哈希值，该哈希值可通过项目提供的工具脚本自动生成。

对于已有条目的更新，例如目标 URL 发生变更或描述信息需要修正，请直接修改对应的 .md 文件内容，并在提交信息中注明更新原因。若仅调整分类标签或补充说明，无需额外通知其他贡献者。

链接校验脚本会定期运行，若发现某一条目指向的 URL 返回 4xx 或 5xx 状态码，该条目将被标记为待复查。贡献者可认领这些待复查条目，确认链接是否永久失效，若失效则删除该条目或在文件中添加弃用注释。

提交变更时请使用常规提交格式，类型包括 feat、fix、docs、chore 等，范围可选 index、scripts、docs。提交信息应简洁明了，例如 docs(index): add new entry for RFC 9110 或 fix(script): handle redirect loop in check_links。

## 常见问题

问：如何查找某个特定主题下的所有索引条目？

答：每个条目文件中包含分类标签字段，您可以在 main 目录下使用 grep 命令结合分类标签进行筛选，例如 grep -r "category: networking" ./main/。此外，项目提供的统计报告脚本 generate_report.py 会按分类输出所有条目的标识符列表，方便您快速浏览。

问：如果目标 URL 发生变更，我需要手动更新所有引用该 URL 的条目吗？

答：不需要。每个条目文件独立存储一个外部链接，若某一条目的目标 URL 发生变更，只需修改该条目文件中的原始链接字段即可。其他文档中通过该条目标识符引用的位置无需任何改动，因为引用的是标识符而非原始 URL。

问：项目是否支持自动检测并修复失效链接？

答：项目内置的链接校验脚本 check_links.py 可以自动检测失效链接并生成报告，但不支持自动修复。对于检测到的失效链接，建议贡献者手动核查目标站点是否临时不可用或已永久迁移。若目标站点已永久迁移，请更新条目文件中的原始链接字段；若站点已彻底关闭，请在条目文件中添加弃用标记。
