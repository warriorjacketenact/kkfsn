# CSU Tech Knowledge Gateway (CSU-TKG)

CSU Tech Knowledge Gateway 是一个面向开发者、技术研究人员与开源爱好者的结构化技术资源导航与知识汇总系统。项目定位于对分散在互联网各处的优质技术文档、教程、工具链说明与工程实践进行系统性收集、分类与索引，帮助用户快速定位特定技术主题下的高价值信息，降低信息检索成本。

项目本身不直接托管大量文本内容，而是以轻量级的 Markdown 链接索引方式运行，通过严格的条目管理与分类体系，将零散的外部资源整合为可检索、可浏览、可维护的知识图谱。目标用户包括正在学习特定技术栈的初中级开发者、需要查阅边缘技术细节的资深工程师，以及希望了解某领域全貌的技术决策者。

## 功能概览

**结构化资源索引**：所有收录的外部链接按照技术领域、资源类型与适用场景进行多级分类，支持快速筛选。

**哈希锚定永久链接**：每个资源条目使用内容哈希作为文件名，确保资源定位的稳定性和可追溯性，避免链接失效导致的检索中断。

**轻量级 Markdown 存储**：所有索引数据以纯 Markdown 文件形式存储在仓库中，无需数据库，便于版本控制和离线阅读。

**分类标签体系**：每条资源附带技术栈标签（如 Python、Rust、Kubernetes）、资源类型标签（教程、规范、工具、论文）和难度标签（入门、进阶、专家）。

**全文检索支持**：借助 GitHub 原生搜索能力，用户可通过关键词在仓库内快速定位包含特定术语的资源条目。

**社区维护机制**：通过 Pull Request 流程接受外部贡献，所有新增资源需经过格式校验与分类审核，保证索引质量。

**批量导入与去重**：支持批量添加资源链接，自动检测重复条目，避免冗余存储。

**多视图浏览**：提供按技术领域浏览、按资源类型浏览和按更新时间浏览三种视图模式，适配不同使用习惯。

## 应用场景

技术选型调研：当团队需要评估多个技术方案（如消息队列选型、前端框架对比）时，可通过本项目的相关分类快速获取各技术的官方文档、社区评测与生产案例链接，集中对比分析。

问题排查参考：开发者在排查特定框架或工具的异常行为时，可在项目中检索相关技术标签，定位到官方 Issue 讨论、社区解决方案与性能调优指南，加快问题定位速度。

学习路径规划：初学者希望系统学习某个技术领域（如云原生、机器学习）时，可依据项目内按难度分类的教程与课程资源，制定从入门到深入的学习路线。

文档离线备份：团队可将常用的内部技术文档与外部参考资料通过本项目进行统一索引和本地克隆，构建团队内部的技术知识库镜像。

开源项目评估：在引入新的开源依赖前，开发者可通过项目内收录的代码仓库链接、技术博客与社区活跃度分析资料，全面评估项目的成熟度与维护状态。

## 快速开始

以下命令可在本地完整克隆并运行本项目的索引服务。

```bash
git clone https://github.com/warriorjacketenact/csutkg.git
cd csutkg
npm install
npm run build
```

执行完毕后，项目将在本地生成静态索引页面，默认监听端口 8080。用户可通过浏览器访问 localhost:8080 进行资源浏览。若只需使用纯 Markdown 索引文件，可直接在 `./main` 目录下查看所有条目，无需执行构建步骤。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | >= 18.0.0 | 用于运行构建脚本与本地开发服务器 |
| npm | >= 9.0.0 | 依赖包管理工具 |
| Git | >= 2.30.0 | 用于克隆仓库和版本控制操作 |
| Python | >= 3.9 (可选) | 仅当需要运行辅助分析脚本时需要 |
| curl | >= 7.68.0 (可选) | 用于批量下载验证资源可用性 |
| jq | >= 1.6 (可选) | 用于处理 JSON 格式的元数据文件 |
| shellcheck | >= 0.7.0 (可选) | 用于校验贡献脚本的语法正确性 |

## 文档导航

| 层面 | 目录 | 回答的问题 |
|------|------|-----------|
| 用户入口 | `./main/` | 所有资源索引条目存放位置，用户按需查阅或搜索具体技术主题 |
| 分类配置 | `./config/categories.json` | 定义技术领域分类、标签体系和资源类型枚举，决定新条目的归类方式 |
| 工具脚本 | `./scripts/` | 包含批量导入、去重检查、格式校验和静态页面生成等自动化工具 |
| 贡献指南 | `./CONTRIBUTING.md` | 外部贡献者需遵循的提交规范、格式要求与审核流程说明 |

## 贡献指南

外部贡献者可通过以下步骤参与本项目的资源索引维护与扩展。

第一步：Fork 本仓库至个人账户，并在本地克隆 Fork 后的副本，同时配置 upstream 指向原仓库以便同步更新。

第二步：在 `./main` 目录下新增资源条目文件，文件命名采用内容哈希值（可通过脚本自动生成），文件内部需包含资源标题、原始 URL、技术标签、分类归属和简短摘要等元数据，格式需严格遵循 `config/schema.json` 中的定义。

第三步：运行本地校验脚本 `./scripts/validate.sh` 检查新增条目的格式合规性，同时执行 `npm run test` 确保未破坏现有功能。若校验失败，需根据错误提示进行修正。

第四步：提交变更并推送到个人 Fork 仓库，随后通过 GitHub 界面发起 Pull Request 至原仓库的 main 分支。PR 描述中需说明新增资源的来源、分类依据和适用场景。

第五步：等待项目维护者审核。审核通过后，PR 将被合并，新增资源正式纳入索引。若审核未通过，维护者会在 PR 中反馈修改意见，贡献者需据此调整后再次提交。

## 常见问题

问：资源条目中的原始外部链接失效了怎么办？

答：项目鼓励用户通过 Issue 报告失效链接。维护者会定期运行链接可用性检查脚本，对失效链接进行标记并尝试寻找替代来源。若无法找到有效替代，该条目将被移至归档目录并添加过期说明。

问：如何查找特定技术领域的所有资源？

答：用户可通过两种方式检索。其一，在 GitHub 仓库的搜索框中输入技术关键词（如 "Kubernetes" 或 "Rust"），搜索范围限定在 `./main` 目录下。其二，使用项目构建生成的静态页面中的分类筛选功能，按预定义的技术领域标签进行浏览。

问：项目是否接受非技术类资源的收录？

答：原则上本项目专注于技术主题，包括但不限于编程语言、框架工具、基础设施、算法理论与工程实践。非技术类资源（如职业发展、团队管理、设计灵感等）不在收录范围内，但贡献者可推荐相关的优秀项目，本项目可为其提供友情链接。
