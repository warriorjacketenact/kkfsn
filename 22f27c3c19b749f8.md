# CSUtkg Resource Indexer

CSUtkg Resource Indexer 是一个面向技术文档与代码资源的外链汇总与结构化索引工具。该项目并非传统的代码库或框架，而是一个持续维护的资源导航系统，用于将分散于多个仓库、文档站点与代码片段中的技术链接统一收录、分类与版本化存档。目标用户包括技术文档编写者、开源项目维护人员、技术调研团队以及需要系统化整理外部参考资料的个人开发者。通过提供统一的元数据描述与快速检索入口，CSUtkg Resource Indexer 旨在解决技术资源分散、链接失效、上下文缺失等常见问题，为上层应用或人工查阅提供稳定、可机读的资源清单。

## 功能概览

**资源快照归档** 对每个收录的外部链接自动提取基础元数据，包括来源仓库、文件路径、提交哈希或时间戳，确保资源可追溯。

**分类标签系统** 支持按技术领域、资源类型、适用场景等多维度标签对链接进行标记，便于后续筛选与聚合。

**版本化索引更新** 每次收录操作均记录版本信息，索引文件本身纳入 Git 版本控制，支持回滚与变更审计。

**多格式输出支持** 索引数据可输出为 Markdown 表格、JSON 结构化文件或纯文本列表，适应不同下游工具链的需求。

**链接健康检查** 提供可选的定期检查机制，对已收录链接进行可达性验证，标记失效或变更的资源。

**全文检索辅助** 基于文件名与目录结构生成关键词映射，支持简单的 grep 或 IDE 内搜索定位。

**增量导入接口** 允许通过 CSV 或 JSON 文件批量导入外部链接清单，自动合并至现有索引。

**权限与可见性分级** 支持将资源标记为公开、内部或受限访问，便于团队协作时控制信息暴露范围。

## 应用场景

技术文档撰写与维护：技术作者在编写教程或 API 文档时，需要引用大量外部规范、示例代码或参考实现。CSUtkg Resource Indexer 可提供经过筛选的稳定链接池，避免直接搜索带来的信息过载。

开源项目依赖梳理：开源项目维护者需要记录项目所依赖的第三方库、工具链文档或社区讨论帖。该索引器可作为依赖说明的补充，提供直达参考资料的快捷方式。

技术调研与竞品分析：调研团队在评估多个技术方案时，需汇总大量官方文档、性能测试报告与案例研究。索引器可按主题分类存储这些链接，并支持导出为共享清单。

个人知识库构建：开发者可将日常阅读的技术博客、演讲幻灯片、规范草案等资源统一归档，配合标签系统实现个人技术知识体系的可检索化。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户建议通过 WSL 或 Git Bash 执行。

```bash
# 克隆仓库
git clone https://github.com/warriorjacketenact/csutkg.git
cd csutkg

# 安装依赖（仅需 Python 3.8+ 与标准库，若需健康检查请安装 requests）
pip install requests --user

# 运行索引构建脚本，生成资源清单
python scripts/build_index.py --input ./collected_links --output ./index.md
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.8 或更高版本 | 是 | 用于运行索引构建与检查脚本，建议使用 3.10 以上以获得更好的类型提示支持 |
| Git 2.20 或更高版本 | 是 | 用于克隆仓库和提交索引更新，推荐 2.30 以上 |
| requests 库 2.25+ | 否 | 仅当启用链接健康检查功能时需要，可通过 pip 单独安装 |
| pytest 7.0+ | 否 | 仅当需要运行单元测试时安装，用于验证索引格式的正确性 |
| Markdown 解析器（如 mistune） | 否 | 用于将索引导出为结构化 HTML 时可选，标准输出为 Markdown 无需此依赖 |
| 磁盘空间 50 MB 以上 | 是 | 用于存储索引文件、缓存元数据及日志，建议预留 200 MB |

## 贡献指南

欢迎提交改进建议与扩展脚本。请遵循以下步骤：

1. 在提交 Pull Request 前，先阅读 docs/submission_standards.md 文件，确保新增链接或脚本变更符合项目规范。

2. 对于新增链接收录，请将链接文件放置于 collected_links 下按日期命名的子目录中，并附带一个简要的 metadata.json 条目，说明来源与标签。

3. 对于脚本功能修改，请在 tests 目录下补充相应的单元测试，确保现有功能不被破坏。运行 pytest 验证所有测试通过后再提交。

4. 更新文档：如果您的变更影响了用户操作流程或脚本参数，请同步更新 docs 目录下对应的手册文件，并确保示例命令可复现。

5. 提交 PR 时，请在描述中清晰说明变更目的、影响范围以及测试结果。维护者将在 3 个工作日内审查。

## 常见问题

Q: 如何批量导入大量外部链接而不逐个手动创建文件？

A: 您可以将链接列表写入一个 CSV 文件（每行一个 URL），然后使用 scripts/import_batch.py 脚本（需自行编写或扩展）自动分割并生成对应的元数据文件。项目目前提供了 import_csv.py 示例，位于 scripts/contrib 目录下，供参考修改。

Q: 健康检查脚本报错 SSL 证书验证失败，如何处理？

A: 如果您处于企业内网或使用自签名证书的环境，可以在 check_links.py 中为 requests.get 调用添加 verify=False 参数。但请注意，关闭验证会降低安全性，仅建议在受信网络中临时使用。生产环境请配置正确的 CA 证书包。

Q: 索引文件中的链接顺序如何确定，能否按字母或标签排序？

A: build_index.py 默认按照收录时间（文件修改时间）升序排列。若要按标签或域名排序，可以在 scripts/utils/formatters.py 中修改 sort_key 函数，目前预留了 sort_by 参数接口，但需自行实现比较逻辑。
