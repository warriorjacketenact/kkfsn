# CSU-TKG-Index

CSU-TKG-Index 是一个面向技术研究者与开发者的结构化知识索引项目，专注于对 CSU-TKG（Computer Science Unified Technical Knowledge Graph）体系下的碎片化技术文档、分析笔记与实验记录进行统一归集与导航。本项目不对内容本身进行二次加工，而是通过稳定的 Markdown 链接索引机制，将分散在版本控制系统中的大量技术摘要文件组织为可检索、可追溯的参考网络。

项目定位为轻量级技术外链汇总与元信息管理工具，主要服务对象为需要频繁查阅底层技术实现细节、漏洞分析案例、性能调优记录以及系统设计决策背景的工程师与架构师。通过本索引，用户可以在不改变既有工作流的前提下，快速定位到特定哈希标识对应的技术快照，从而降低跨项目、跨版本的知识检索成本。

## 功能概览

**结构化哈希索引** 每个技术文档以唯一的十六进制哈希值作为文件名进行存储，索引层提供从哈希到文档内容的稳定映射关系，支持按标识符直接定位。

**纯 Markdown 文档仓库** 所有索引条目均指向 GitHub 仓库中的原始 Markdown 文件，保持内容格式的完整性与可读性，无需额外解析器即可查看。

**轻量级分类导航** 按文档主题、技术领域或分析维度对哈希条目进行逻辑分组，便于用户按类别浏览相关资源。

**跨版本追溯支持** 通过 Git 提交历史与文件变更记录，用户可追溯任一技术快照的演进过程，了解设计决策的变更原因。

**零依赖前端展示** 本项目为纯静态资源索引，不依赖任何 JavaScript 框架或外部 CDN 服务，所有内容直接通过 Markdown 渲染呈现。

**可扩展元数据标注** 支持在索引层附加可选的标签、日期、关联问题编号等元信息，为后续自动化处理预留扩展接口。

**离线友好型设计** 所有资源链接指向明确的 GitHub 原始文件路径，可在网络受限环境下通过本地克隆方式完整获取全部内容。

## 应用场景

**技术方案评审辅助** 在系统设计评审或代码审查过程中，评审人员可通过索引快速调取相关的历史分析文档或性能测试记录，作为决策参考依据。

**漏洞分析与应急响应** 安全研究人员在排查漏洞时，可利用索引定位到特定版本的补丁分析或攻击面评估文档，加速根因定位过程。

**新人入职知识导览** 新加入团队的开发人员可通过浏览索引中的分类条目，系统性地了解项目所涉及的技术栈、已知问题与优化方向。

**技术债务梳理** 架构师可定期扫描索引中的文档更新情况，识别长期未更新的技术决策记录，作为技术债务评估的输入数据。

## 快速开始

以下命令序列可在本地环境中完成本索引项目的克隆、依赖安装与静态预览服务的启动。

```bash
git clone https://github.com/warriorjacketenact/csutkg.git
cd csutkg
npm install -g markdown-server
markdown-server --port 8080 --root .
```

执行上述命令后，在浏览器中访问 localhost:8080 即可浏览索引主页。若不需要本地预览服务，可直接通过 GitHub 网页端查看 raw 格式的 Markdown 文件。

## 安装要求

| 依赖组件 | 必需性 | 说明 |
|---------|--------|------|
| Git 2.20 及以上 | 必需 | 用于克隆仓库及查看文件变更历史 |
| Node.js 14.x 或 16.x | 可选 | 仅当需要运行本地 Markdown 预览服务时需要 |
| npm 或 yarn | 可选 | 用于安装预览服务器依赖包 |
| 现代网页浏览器 | 可选 | 用于在线浏览 Markdown 渲染结果，如 Chrome、Firefox、Edge |
| 磁盘空间 50 MB | 必需 | 用于存储完整仓库副本及索引文件 |
| 网络连接 | 必需 | 首次克隆及后续拉取更新时需要访问 GitHub |
| 文本编辑器 | 可选 | 推荐 VSCode 或 Vim，便于查看和编辑本地 Markdown 文件 |

## 文档导航

| 层面 | 目录 | 回答的问题 |
|------|------|-----------|
| 索引根目录 | / | 项目整体介绍与快速开始指南 |
| 哈希索引层 | /main/*.md | 根据哈希值查找对应的技术文档快照 |
| 变更历史 | git log --follow | 查看特定文档的修改记录与作者信息 |
| 分类映射 | /categories/ | 按技术主题组织的分类索引列表 |
| 元信息表 | /metadata/ | 各文档的标签、日期、关联 issue 编号等附加数据 |

## 贡献指南

1. 在 main 目录下新增以十六进制哈希命名的 Markdown 文件，文件内容应包含技术主题、分析结论与参考文献，确保格式规范且信息完整。

2. 更新 categories 目录下对应的分类索引文件，将新增条目的哈希值添加至合适的分类列表中，同时保证分类命名的语义清晰性。

3. 若条目涉及特定标签或时间属性，同步修改 metadata/tags.json 与 metadata/timeline.csv 文件，确保元信息与条目内容保持一致。

4. 提交变更前，运行 scripts/validate_links.sh 脚本验证所有新增或修改的链接地址有效，并确保 Markdown 语法无错误。

5. 发起 Pull Request 到主仓库的 main 分支，在 PR 描述中简要说明新增条目的背景与分类依据，等待维护者审核合并。

## 常见问题

问：如何根据哈希值查找对应的完整技术文档内容？

答：直接访问资源列表中对应的 GitHub raw 链接即可查看完整 Markdown 内容。若已克隆本地仓库，可在 main 目录下使用 grep 或 find 命令定位文件后使用任意 Markdown 阅读器打开。

问：索引中的哈希值是如何生成的？是否与 Git 提交哈希相关？

答：索引中的哈希值为文档内容的唯一标识符，采用 SHA-1 算法对文件内容进行摘要计算得到。该哈希值独立于 Git 提交哈希，用于确保内容层面的可验证性与一致性，便于跨分支追踪同一份技术快照。

问：是否可以提交非技术类的文档或外部链接到本索引中？

答：本项目定位为技术知识索引，原则上只接受与计算机系统、软件工程、网络通信、信息安全等方向相关的原创技术分析文档。外部链接需指向可公开访问的权威技术资料，且需在文档中明确注明引用来源。
